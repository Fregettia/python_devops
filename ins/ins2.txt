"""
1 核心目标：设备管理基础设施 
    云端设备管理服务 
    边端设备基础程序 
    云端协同交互协议 
2 基本要求：云端设备管理服务 
2.1 实体定义 
2.1.1 边端设备 
    基本信息：设备编号、设备名称、设备描述、创建时间、更新时间。 
    硬件信息：硬件序列号、硬件型号、以太网卡MAC地址、WiFi网卡MAC地址、LTE IMEI编码（SIM卡唯一标识）。 
    认证信息：Salt、Secret（Password & Salt 散列）。 
    软件信息：基础程序版本号、基础程序上次更新时间、基础程序状态、业务程序版本号、业务程序上次更新时间、业务程序状态。 
2.1.2 设备分组 
    基本属性：分组编号、分组名称、分组描述、创建时间、更新时间 
2.1.3 状态日志 
    CPU信息：如user cpu time、system cpu time、idle cpu time、io wait cpu time等 
    内存信息：如内存总量、空闲内存、buff/cache总量等 
    硬盘信息：如分区磁盘大小、根目录所占空间比等 
2.2 接口要求 
    北向和南向接口均要求使用 TLS 自签发证书加密。 
2.2.1 北向接口 
    北向接口主要面向顶层业务提供边端设备管理服务相关接口，以 HTTP 作为通信协议，以 HTTP REST-like API 接口形式暴露服务。 
    接口需求： 
    1,接口服务,功能备注,
    2,设备注册(创建设备),"注册设备,至少提供随机密钥",
    3,基于设备编号删除设备,N/A,
    4,基于设备编号查询设备,N/A,
    ...
2.2.2 南向接口 
    南向接口主要面向边端设备提供边端设备对接服务相关接口，以 Web Socket 作为通信协议，以函数 接口形式暴露服务。 
    要求采用JSON-RPC构建应用层通信协议。 
    接口需求： 
    1,接口服务,功能备注
    2,执行设备登录验证,面向设备基础程序执行设备登录验证
    3,接收设备状态数据,接收设备基础程序所上传的状态数据
    4,下发设备指令操作,"向设备基础程序下发基础设施类指令, 如重启设备reboot,或者通过sftp将本地日志上传到指定目录。"
3 基本要求：边端设备基础进程 
    1,功能要求,功能备注
    2,远程登录设备管理服务,使用WebSocket成功建立连接后立刻提交鉴权信息
    3,定时上传设备状态数据,通过WebSocket定时向设备管理服务上传设备状态
    4,记录设备本地操作日志(JSON文件),触发事件至少包括:
        1.发起连接、2通道建立成功、3鉴权结果、4周期性上报数据
    5,执行云端基础设施类捐令,执行指令至少包括:
        1. reboot (root权限执行客户端程序)
        2.通过sftp将日志上传到捐定目录
"""
